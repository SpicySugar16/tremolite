# 代码审查 (Code Review)

审查代码变更的系统性方法，覆盖逻辑正确性、安全性、性能和风格。

## 审查看点

1. **逻辑正确性** — 边界条件、竞态条件、错误处理路径、空值安全
2. **安全性** — 注入攻击、权限绕过、敏感数据泄漏、未验证的输入来源
3. **性能** — 不必要的循环/n+1查询、内存分配模式、大对象复制、缓存缺失
4. **可维护性** — 命名清晰度、函数复杂度(SLOC)、重复代码、测试覆盖率缺口
5. **兼容性** — API 向后兼容、配置迁移、依赖版本冲突

## 输出格式

每条问题标记严重级别：
- 🔴 **阻塞** — 会导致数据丢失/安全漏洞/生产事故
- 🟡 **重要** — 可能造成不稳定或后续维护困难
- 🔵 **建议** — 风格/可读性/规范的小问题

## 原则

- 先通读变更上下文（Issue/PR描述），再逐行审查
- 指出问题就给出修改建议，不止说「这不对」
- 对不熟悉但看似正确的代码，标记「需手动测试确认」
