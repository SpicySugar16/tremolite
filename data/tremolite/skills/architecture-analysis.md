# 架构分析 (Architecture Analysis)

对软件项目的整体架构进行系统分析，理解模块划分、数据流和部署拓扑。

## 分析方法

1. **模块划分** — 识别 crate/module/package 边界，评估内聚性与耦合度
2. **数据流追踪** — 请求入口 → 处理链路 → 存储/外部调用 → 响应出口
3. **依赖分析** — 编译时依赖（Cargo.toml/Cargo.lock）和运行时依赖（外部服务、DB）
4. **部署拓扑** — 进程模型、网络连接、状态存储方式、水平扩展路径
5. **配置与可观测** — 配置方式（文件/env/远程）、日志/指标/链路追踪覆盖

## 输出格式

- 架构总览图（模块/数据流/部署三个视图）
- 关键设计决策及理由（ADR）
- 设计模式识别和使用情况
- 技术债务定位

## 原则

- 先看 README/CONTRIBUTING/docs，再读源码
- 从外部（CLI入口/HTTP端点）向内部（核心逻辑）推进
- 修改件比源文件更值得关注（模块间连接处的问题比模块内部多）
